# deepin Agent Teams RFC

仓库代号暂为 `deepin_assistant`，进阶任务正式交付名为 `deepin Agent Teams`。

## 1. 项目信息

- 赛道：`飞桨黑客松第十期 文心合作伙伴赛道`
- 厂商：`统信`
- 任务编号：`27`
- 任务名称：`deepin Agent Teams 智能体团队协作系统`
- 仓库：`https://github.com/guanzi008/deepin_assistant`

## 2. 目标

在 `deepin 25` 上实现一个具备环境感知、意图识别、多智能体协同和本地工具调用能力的桌面助手，并完整演示：

1. 智能邮件助手
2. 系统问题智能诊断与修复

## 3. 单一交互入口

应用采用统一图形入口：

- 右侧悬浮助手面板

支持：

- 自然语言输入
- 主动建议
- 执行预览
- 用户确认
- 修复结果验证

## 4. 智能体设计

### 4.1 信息收集员

- 采集活动窗口标题、应用名、剪贴板、截图和系统状态
- 聚合多源上下文，输出结构化任务上下文

### 4.2 系统操作员

- 进行意图判断
- 执行问题澄清
- 生成修复步骤
- 调用本地工具

### 4.3 内容创作员

- 撰写邮件
- 汇总系统诊断结果
- 生成面向用户的结果说明

### 4.4 结果验证员

- 校验动作效果
- 给出回退或替代建议

## 5. 环境感知

上下文来源：

- 活动窗口元数据
- 剪贴板
- 定向截图 + OCR
- 系统日志与服务状态
- 文件系统检索
- 设备与网络状态

原则：

- 低成本信号优先
- 截图按需触发
- 敏感操作前必须确认

## 6. 模型方案

至少调用两款文心模型 API，通过 `erniebot SDK` 接入：

- 轻量模型：负责意图分类、路由、是否需要追问
- 强模型：负责邮件撰写、诊断推理、修复计划生成

## 7. 核心场景

### 7.1 智能邮件助手

输入：

- 当前窗口
- 剪贴板内容
- 相关文档摘要

输出：

- 收件人建议
- 邮件标题
- 邮件正文
- 发送前预览

### 7.2 系统问题诊断与修复

优先覆盖：

- 打印机连不上
- 没有声音了
- 帮我安装微信
- 网络异常
- 软件源异常

闭环：

- 问题输入
- 系统采集
- 方案生成
- 用户授权
- 修复执行
- 结果验证

## 8. 技术路线

- 前端：`React + Vite`
- 本地服务：`Node.js`
- 系统工具：`systemctl / journalctl / lpstat / nmcli / apt / 文件检索`
- 邮件：草稿导出 + SMTP 发送

## 9. 当前基础

已完成原型能力：

- 图形界面
- 实时诊断
- 执行控制台
- 支持包 / 工单导出
- 高权限动作授权回执

## 10. 下一步实施

1. 增加邮件场景上下文采集与草稿生成
2. 增加活动窗口、剪贴板、定向截图感知
3. 扩展系统问题场景，减少对单一打印参数调优的依赖
4. 在 deepin 25 真机完成演示录制与截图采集

## 11. 验收映射

- 系统感知能力：多源上下文采集与融合
- 多智能体协同：至少 3 个角色协作
- 场景演示：邮件 + 系统修复两个闭环
- 文档与报告：源码、部署文档、技术报告、演示脚本齐备

## 12. 结论

该方案范围可控，贴合统信赛题要求，且已具备原型基础，可在锁定任务后快速推进为可演示、可复现的完整作品。
