# 演示脚本

## 场景一：悬浮启动器

1. 启动桌面程序后，先看到右下角的悬浮按钮。
2. 点击悬浮按钮，唤起或收起右侧主面板。
3. 说明这个入口不是装饰，而是整个桌面助手的常驻唤起点。

## 场景二：侧边面板

1. 点击悬浮按钮打开侧边面板。
2. 先看当前系统快照：
   - 系统版本
   - 根分区和内存
   - 网络状态
   - 打印队列
   - 音频状态
   - 包管理状态
3. 点击“刷新诊断”，说明这些内容来自当前机器，而不是写死的演示数据。
4. 切换一个问题，比如：
   - 打印机连不上
   - 没有声音
   - 安装失败
5. 展示系统先给出的判断、风险等级和处理顺序。

## 场景三：邮件草稿和截图

1. 切到邮件助手入口。
2. 现场输入一段邮件素材，或者从文档里复制一段内容。
3. 展示系统整理出来的邮件主题、正文和附件说明。
4. 打开截图采集和导出入口，说明这次会把桌面状态一起带进来。
5. 展示导出的草稿和截图文件，强调这些不是只在面板里看一眼就结束。

## 场景四：打印修复

1. 进入打印修复链。
2. 先看旧队列、驱动包和 CUPS 过滤链状态。
3. 依次执行：
   - 删除旧队列
   - 重装关键驱动包
   - 修复 CUPS 过滤链权限
4. 对需要高权限的动作，展示系统导出的待执行脚本和 `pkexec` 命令。
5. 说明每次执行都会留下动作日志和回执，后面还能继续追查。

## 场景五：常见问题

1. 切到网络、音频或安装入口。
2. 展示系统采集到的本地状态。
3. 选择执行重启网络服务、重启音频会话或修复包管理状态。
4. 展示动作结果、导出文件和执行日志。

## 场景六：打包和安装

1. 说明项目用 `CMake` 构建。
2. 说明输出是 `CPack` 生成的 `tar.gz`。
3. 给出二进制、`.desktop` 启动入口、日志目录和草稿导出目录所在位置。
4. 说明既能直接解包运行，也能安装到当前用户目录。
